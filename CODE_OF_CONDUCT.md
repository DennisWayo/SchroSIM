# Code of Conduct

## Our Commitment
We want SchroSIM to be a professional, inclusive, and constructive community for students, researchers, and engineers. Contributors and maintainers are expected to act with respect, accountability, and technical integrity.

## Expected Behavior
Examples of positive behavior include:
1. Giving constructive, actionable feedback.
2. Respecting different technical viewpoints and experience levels.
3. Owning mistakes and correcting them.
4. Focusing discussions on facts, evidence, and project outcomes.

## Unacceptable Behavior
Examples of unacceptable behavior include:
1. Harassment, threats, or discriminatory language.
2. Personal attacks, insults, or sustained hostile behavior.
3. Doxxing or sharing private information without permission.
4. Disruptive conduct that prevents productive collaboration.

## Scope
This Code of Conduct applies in all project spaces, including issues, pull requests, discussions, commits, and any public interactions representing this project.

## Reporting
If you experience or witness unacceptable behavior:
1. Report privately to project maintainers using GitHub private communication channels where available.
2. Contact the maintainer account at https://github.com/DennisWayo and request a private follow-up channel.
3. If a private channel is not available, open a minimal public issue requesting private follow-up without posting sensitive details.

Maintainers will review reports promptly, handle them as confidentially as possible, and take appropriate corrective action.

## Enforcement
Maintainers may remove or edit comments, commits, code, issues, or other contributions that violate this Code of Conduct. Repeated or severe violations may result in temporary or permanent exclusion from project participation.
